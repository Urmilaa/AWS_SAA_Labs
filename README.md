# ☁️ AWS Certified Solutions Architect – Associate (SAA) Exam Guide

A comprehensive guide to understanding the **AWS Solutions Architect Associate exam pattern** and a **proven preparation strategy** to help you succeed.

---

| Feature             | Details                             |
| ------------------- | ----------------------------------- |
| Certification Level | Associate                           |
| Exam Duration       | 130 minutes                         |
| Total Questions     | 65                                  |
| Question Types      | Multiple Choice & Multiple Response |
| Passing Score       | ~720 / 1000                         |
| Exam Cost           | $150                                |
| Mode                | Online / Test Center                |

---

## 🧠 Question Pattern

### ✅ 1. Multiple Choice Questions (MCQ)

* Single correct answer
* Tests core AWS concepts

### ✅ 2. Multiple Response Questions

* “Select TWO/THREE correct answers”
* No partial marking (all correct required)

### 🔥 Exam Tip

Focus on selecting answers that are:

* Cost-effective
* Highly available
* Fully managed (serverless preferred)

## 🧪 How Questions Are Framed

The exam is **scenario-based**. You’ll be asked to choose the **best architecture solution** based on requirements like:

* High Availability
* Fault Tolerance
* Cost Optimization
* Performance

### Example Scenario:

> A company needs a scalable, highly available architecture with minimal operational overhead.

👉 You must decide:

* Multi-AZ vs Multi-Region
* Managed vs self-managed
* Cost vs performance trade-offs

---

## 🚀 Preparation Strategy

### 🟢 Step 1: Build Strong Fundamentals

Focus on:

* VPC (Subnets, Routing, NAT, IGW)
* Storage (S3, EBS, EFS)
* Compute (EC2 vs Lambda)

---

### 🟢 Step 2: Develop Architecting Mindset

Always think in terms of:

* High Availability → Multi-AZ
* Fault Tolerance → Auto Scaling
* Cost Optimization → Lifecycle policies, Spot instances
* Security → Least privilege (IAM)

---

### 🟢 Step 3: Hands-on Practice (Critical)

Perform labs such as:

* Launch EC2 and attach EBS
* Host static website on S3
* Configure Load Balancer + Auto Scaling
* Setup RDS Multi-AZ

---

### 🟢 Step 4: Practice Questions

* Solve 300–500 practice questions
* Analyze why incorrect answers are wrong

---

### 🟢 Step 5: Revision Strategy

Prepare short notes on:

* Service limits
* Key differences (EBS vs EFS vs S3)
* Use cases

---

## ⚠️ Common Mistakes to Avoid

### ❌ Ignoring Keywords

Look for terms like:

* Highly available
* Cost-effective
* Serverless

---

### ❌ Choosing Overly Complex Solutions

* Prefer managed services over custom setups

---

### ❌ Not Reading Carefully

* Understand what is being asked:

  * “MOST cost-effective”
  * “LEAST operational overhead”

---

## 🧠 Golden Rules for the Exam

### 🏆 Rule 1: Managed > Self-managed

Example: Lambda over EC2 (when possible)

### 🏆 Rule 2: Multi-AZ for High Availability

### 🏆 Rule 3: Use S3 for Durability

### 🏆 Rule 4: Decouple Architectures

Use services like SQS and SNS
